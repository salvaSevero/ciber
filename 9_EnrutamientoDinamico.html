    Protocolos de enrutamiento dinámico.¶

    Objetivos.

    • Descubrir automáticamente las diferentes rutas que puede haber para llegar a un destino.

    • Elegir la mejor ruta hacia el destino llegado el caso, por ejemplo aquel más rápido, menos saturado, etc.

    • Si cae un camino el protocolo de enrutamiento dinámico lo detecta y descarta dicho camino.

    ¿Cómo funcionan los protocolos de enrutamiento dinámico?

    Los protocolos de enrutamiento trabajan con las tablas de rutas y por tanto se configuran en los routers. Los routers a los que se le configura el protocolo establecen una comunicación entre sí, esto les permite intercambiarse la tabla de rutas o estados de ciertos caminos que pueden haber sufrido una avería.

    ¿Qué es el Tiempo de conversión?

    La convergencia se produce cuando todos los routers que conforman el protocolo de enrutamiento dinámico tienen toda la información completa. Mayormente es enviar la información de su tabla de rutas, recibir la información del resto, añadirla y establecer cuál es la mejor ruta activa en ese momento para cada destino.

    La red no es operativa hasta que el protocolo de enrutamiento dinámico en cuestión haya convergido. Es decir, si sufrimos una avería grande y el protocolo de enrutamiento dinámico cae, una vez restablecido la conexión debemos esperar a que el protocolo de enrutamiento converja completamente para volver a tener conectividad, obviamente este tiempo también depende del número de rutas que se compartan.

    En orden de velocidad de convergencia de los protocolos de enrutamiento dinámico encontraríamos: ISIS, EIGRP, OSPF y BGP.

    Protocolos de enrtutamiento dinámico interno (IGP) y externo (EGP).

    IGP:

    • Sirven para comunicarse con routers dentro de tu red.

    • No podemos anunciar rutas a Internet a través de estos protocolos.

    • Habitualmente propagan las redes privadas de una empresa.

    • ISIS, OSPF e EIGRP son los más usados, especialmente OSPF e EIGRP.

    EGP:

    • Son aquellos protocolos que permiten anunciar rutas entre AS (Sistemas Autónomos) diferentes.

    • Permiten anunciar rutas a Internet.

    • El protocolo usado actualmente es BGP.

    Sistema Autónomo

    Un AS es una entidad bajo la cual existen una o varias redes. Se puede ver como un identificador para una empresa.

    OSPF.¶

    Se elige un router maestro, conocido como “DR o router designado”, además se elige un router de backup o BDR.

    La información con las rutas de cada router se conocen como LSA.

    El DR es el encargado de enviarle el LSA de todo el conjunto de routers al resto de vecinos asociados al protocolo, esto se hace mediante la IP reservada multicast 224.0.0.5.

    Un vecino envía su LSA únicamente al DR y BDR y éstos serán los que se encargarán a posteriori de enviárselo a los demás. Esto permite un mejor control de que todos los routers tengan la misma tabla rutas en todo momento, pero por el contrario el DR sufre más carga de trabajo que el resto de routers, por lo que se recomienda que sea un equipo potente.

    Ejemplo de configuración básica.¶

    Configuramos la interfaz/subinterfaz

    interface FastEthernet0/0.5
 encapsulation dot1Q 5
 ip address 192.168.5.1 255.255.255.0
 ip ospf authentication-key salva
 ip ospf dead-interval 5
 ip ospf hello-interval 1
 ip ospf priority 255

    • ip ospf authentication-key salva: Contraseña para que no cualquier router conectado a la red pueda asociarse al protocolo y tener nuestra table de rutas.

    • ip ospf dead-interval X: Segundos tras los cual si no contesta un vecino se le da por muerto y se anulan las rutas que pasan por él. Por defecto 40 segundos.

    • ip ospf hello-interval X: Segundos tras los que manda un saludo a sus vecinos con el fin de comprobar si ha habido alguna avería. Por defecto su valor es 10 segundos.

    • ip ospf priority X: Prioridad de cada router para ser el router designado (DR), el router designado de reserva (BDR) o un simple nodo. Si la prioridad es 0 jamás podrá ser el router designado. A mayor número mayor prioridad (255).

    Para que el protocolo levante todos los routers configurados deben tener los mismos intervalos y la misma contraseña.

    Configuramos el proceso OSPF.

    router ospf 123
 router-id 192.168.5.1
 redistribute connected subnets
 network 192.168.5.0 0.0.0.255 area 1

    • router-id: Es la IP con la que el router se identificará con sus vecinos. Normalmente es la IP que tiene asignada en la interfaz/subinterfaz donde hemos configurado el proceso.

    • redistribute connected subnets: Indica que va a redistribuir todas las redes y subredes que conozca que estén conectadas directamente.

    • network 192.168.5.0 0.0.0.255 area 1: Activa el proceso OSPF en las interfaces donde tenga esa red configurada.
