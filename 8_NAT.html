    NAT (Network Address Translation).¶

    Aproximadamente existen 3706452992 direcciones públicas IPv4 que pueden usarse para navegar por Internet. Si cada dispositivo conectado a Internet usara una de estas direcciones no habría direcciones suficientes.

    Por ello existen 2 soluciones principales:

    • IPv6

    • NAT

    Tipos de NAT.¶

    NAT estático.¶

    Es una traducción de una IP interna a una IP externa. No ahorra IPs, lo único que hace es enmascarar la IP original que hace la petición.

    NAT dinámico.¶

    Dada una bolsa de IPs externas vamos asignando de forma dinámica traducciones a las IPs internas que los soliciten. El ahorro de IPs es mínimo y nulo si el número de IPs externas es igual a de internas.

    NAT por puertos (PAT).¶

    Una IP tiene un total de 65535 puertos. NAT PAT va asignando puertos a cada conexión interna que se solicite, de esta forma con una única IP se consigue dar servicios a múltiples IPs, hasta un total de 65535 conexiones por IP externa, aunque una IP interna hace uso de cientos de conexiones diferentes.

    CGNAT.¶

    En este caso es el operador el que gestiona el NAT de sus clientes de forma centralizada y no el router de cada cliente. Es más eficiente que el NAT PAT. El problema es que el cliente pierde la libertad de poder abrir sus puertos para iniciar un servicio.

    NAT64.¶

    Es un NAT que permite comunicar equipos con IPv6 navegar como si tuvieran asignados IPv4.

    Limitaciones.¶

    A medida que vamos navegando esta tabla NAT del router crece.

    Esto gnera una serie de limitaciones:

    • En este caso toda nuestra LAN está siendo traducida a una IP de la WAN relacionada con sus puertos, pero hay un número de puertos finitos que pueden asociarse a una IP, en caso de que se llenen empezaría a dar problemas. La solución sería asignar un rango de IPs para la traducción en la WAN, esto solo pasaría si tuvieramos una red LAN muy grande que está siendo traducida.

    • Existe también un máximo de memoria destinado a almacenar IPs traducidas, si esto se llena también existe un problema. La solución sería limpiar la tabla NAT o bajar el tiempo que se almacena una conexión en memoria.

    ¿Hay alguna forma para las operadoras de sacar más rendimiento a las direcciones IP públicas en relación al número de clientes que tiene?

    RESPUESTA

    Sí, usando CGNAT.

    • ¿Qué problema puedo tener detrás de un NAT si quiero levantar un servicio para mis clientes? ¿Cuál es la solución? ¿Y si estoy detrás de un CGNAT?
    RESPUESTA

    Si estoy detrás de un NAT el problema es que desde el exterior no se puede iniciar una comunicación, siempre se tiene que iniciar la comunicación desde dentro, por lo que si tenemos un servidor en un NAT no podría ser accedido por los clientes.

    La solución es "abrir los puertos" en el router. Esto no es más que decirle por ejemplo que todo lo que le llegue a la IP pública en el puerto X, por ejemplo el 443, que lo redirija a la IP privada de nuestro servidor.

    Si estamos detrás de un CGNAT no podríamos abrir puertos y nuestra única solución sería llamar a nuestro ISP para que nos sacara del CGNAT.
