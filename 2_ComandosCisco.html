    Comandos básicos Cisco.¶

    Niveles de configuración.¶

    En Cisco existen varios niveles de configuración.

    • Modo Usuario.

      💥 Modo lectura que nos permite consultar información o estado del equipo.

    • Modo Privilegiado.

      💥 Es un modo que permite ver la configuración del equipo y acceder al modo configuración.

      💥 Se accede a este modo mediante el comando “enable”.

    • *Modo configuración.

      💥 Es el modo que nos permite realizar cambios.

      💥 Es importante tener en cuenta que en Cisco los cambios se aplican al instante.

      💥 Se accede desde el modo enable, mediante el comando “configure terminal”

    Autocompletado y Autoayuda¶

    • En el terminal podemos hacer uso del “?” para ver que parámetros u opciones tiene cada comando.

    • Podemos hacer uso del “tabulador” para autocompletar un comando.

    • No es necesario completar el comando para que se ejecute. Por ejemplo en vez de enable podemos ejecutar “en”, en vez de configure terminal “conf t”, en vez de write usar “wr”.

    Configurar Interfaces.¶

    Cada fabricante nombra sus interfaces de diferente forma. En el caso de Cisco los más habituales son los siguientes:

    • Fa: FastEthernet (100Mbps)

    • Gi: GigaEthernet (1Gbps).

    • Te: TenGigabitEthernet (10Gbps).

    Muchos equipos de red son ampliables por módulos, es decir, que podemos tener un switch de 12 puertos y un hueco para insertarle otro módulo de 12 puertos. Otros equipos son stackables, se pueden unir entre si y funciona como un mismo equipo.

    Pues bien, las interfaces nos dan información de donde se encuentra el puerto dentro del equipo.

    Por ejemplo:

    💥 Gi0/0 --> Indica que la interfaz es la primera del primer módulo.

    💥 Gi1/0/3 --> Puede indicar el tercer puerto, del primer módulo, del segundo switch en stack.

    Mostrar Configuración.¶

    Los equipos Cisco tienen dos memorias, una volátil (NVRAM) y otra no volátil (memoria flash).

    Cuando realizamos un cambio en la configuración, este se aplica y dicha configuración se almacena en la NVRAM. Es decir, en caso de reiniciar el equipo perderíamos esos cambios realizados. A esta configuración se le llama “Running Configuration”.

    Cuando el equipo Cisco arranca coge la información de la memoria flash (no volátil). A esta configuración se le conoce como “Startup Configuration”.

    Guardar Cambios.¶

    Si queremos que unos cambios persistan a un reinicio debemos copiar la “Running Config” a la “Startup Config”, es decir, pasarlo de la NVRAM a la memoria flash. Para ello usamos el comando “write” o “wr”.

    Resumen¶

    Comando
    Función
    enable Acceder al modo administrador.
    configura terminal Entrar al CLI para modificar configuración.
    show running-config Ver la configuración actual.
    show interfaces description Información de las interfaces.
    wr Guardar cambios.
