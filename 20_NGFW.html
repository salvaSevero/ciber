    NGFW (Next Generation Firewall).¶

    Los firewall NGFW tienen funcionalidades de hasta capa 7, mientras que los firewall tradicionales solían tener limitaciones en la capa de transporte.

    Los NGFW pueden bloquear por ejemplo por aplicación mientras que los firewall tradicionales estaban limitados a bloquear por IP/Puerto/Protocolo.

    Funcionalidades.¶

    • Clasificación e inspección de protocolos de capa de aplicación.

    • Filtrado basado en listas negras y categorías.

    • Detección y prevención de intrusos (IDS).

    • Inspección de tráfico cifrado https (HTTPS Inspection).

    • Antivirus.

    • Configuración automática de Proxy (WPAD).

    • Control de acceso por MAC.

    • ...

    Fabricantes.¶

    Existen varios fabricantes que tienen firewalls NGFW:

    • PaloAlto.

    • Fortinet.

    • Sophos.

    • CheckPoint.

    • WatchGuard.

    • OPNsense (libre).

    Práctica OPNsense.¶

    Esquema de Laboratorio.¶

    Importante que la vtnet0 sea la que esté conectada a la WAN y la vtnet1 a la LAN.

    Acceso.¶

    Se puede acceder desde la LAN (KALI Linux) a la IP 192.168.200.1 a través de un navegador con las credenciales root / opnsense

    Es recomendable cambiar la resolución del KALI a la misma que la máquina anfitriona.

    Antes de empezar asegúrate que eres capaz de navegar desde el KALI Linux

    Cortar acceso a web (HTTP/HTTPS) por IP.¶

    Ahora vamos a ver cómo crear una regla que deniegue el acceso a un servicio de Internet, como por ejemplo el acceso http y https a www.leagueoflegends.com.

    Pregunta 1

    Realiza una consulta dns (con nslookup desde Windows o host/dig desde Linux) desde la máquina cliente para consultar la IP o direcciones IP de www.leagueoflegends.com. Adjunta una captura con las direcciones IP que devuelve el comando de búsqueda dns.

    • Una vez hemos obtenido las direcciones IP, crearemos un alias con el nombre LOL_COM, de forma que podamos referenciarlo desde las reglas sin tener que repetir las direcciones IP cada vez en varias reglas.

    Para ello vamos a Cortafuegos ➡️ Aliases ➡️ ➕ y configuramos los siguientes parámetros:

    Nombre: LOL_COM (por ejemplo, es descriptivo)

    Escribir: Host(s)

    Contenido: (escribimos las IP, pulsando tabulador después de cada una)

    Descripción: Direcciones IP de www.leagueoflegends.com

    • Ahora creamos un alias para los servicios HTTP y HTTPS.

    AVISO

    Los cambios no se aplican automáticamente debemos pulsar sobre el botón rojo Aplicar.

    Cortar servicios.¶

    A continuación vamos a aplicar una política restrictiva de seguridad a los clientes de la LAN, en la que sólo se va a permitir que se pueda navegar por Internet (http y https) y lanzar pings. El resto de servicios se van a denegar (ftp, ssh, pop3, smtp, etc.).

    Pregunta 4

    A partir de las explicaciones anteriores, crea una regla para permitir el tráfico HTTP y HTTPS hacia cualquier destino desde la LAN y otra que permita el ICMP (ping y traceroute) hacia cualquier destino. Ambas reglas debes crearlas detrás de la regla que deniega navegar por www.leagueoflegends.com. Adjunta pantallazo de todas las reglas.

    Pregunta 5

    Crea otra regla para el servicio DNS (UDP y TCP) pero únicamente que cuyo destino sea el propio firewall.

    Recuerda que el firewall se asigna asimismo como servidor DNS a los clientes mediante DHCP.

    Así evitamos que puedan usar DNS externos a la organización o que alguien se instale un servicio DNS dentro de nuestra LAN.

    Pregunta 6

    Deshabilitaa la regla que permite todo el tráfico desde la LAN hacia cualquier destino. Para ello, edítala y marca Deshabilitar esta regla.

    Para comprobar que no funcionan otros servicios como el ftp puedes probar a lanzar el comando ftp ftp.rediris.es en un terminal y ver si te deja conectar.

    Bloquear acceso a la web por dominio.¶

    Denegar el servicio de un dominio bloqueando la IP tiene mayormente dos desventajas:

    • En una misma IP pueden alojarse varios dominios (ViewDNS).

    • Si el día de mañana por cualquier motivo el dominio cambia de IP nuestro bloqueo dejaría de ser efectivo.

    Por tanto en esta sección vamos a bloquear el dominio sobreescribiendo la petición DNS.

    Servicio ➡️ Unbound DNS ➡️ Sobrescribir
