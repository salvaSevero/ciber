    STACK & LACP.¶

    STACKING.¶

    La posibilidad de hacer stack con ciertos switches permite que múltiples equipamientos puedan interconectarse entre sí, siendo gestionados y funcionando como un mismo equipo.

    Para hacer un stack debemos contar con switches del mismo fabricante y cuyos modelos sean compatibles entre sí para hacer el stack. También deben tener la misma versión de sistema operativo (IOS en Cisco).

    Cada switch tiene un rol:

    • Activo: Es el switch que se encarga de manejar la configuración y hacerla efectiva en el resto de switches.

    • Standby: Es el siguiente switch en la cadena de mando, es decir, si el switch Activo cae, él se convertirá en el activo.

    • Member: Es un switch del stack.

    Los switches se suelen conectar por unos puertos especiales para ello, la interconexión entre estos puertos puede ser la que queramos. No obstante, es muy recomendado seguir el siguiente formato de interconexión para tener la mayor redundancia y disponibilidad.

    LACP (Link Aggregation Control Protocol - IEEE 802.3ad).¶

    • El principal objetivo de este protocol es la creación y gestión de un LAG (Link Aggregation Group). Un LAG permite que varias interfaces de red funcionen como uno solo, de forma que incrementamos el ancho de banda y la redundancia, pues si un enlace cae, el grupo sigue funcionando (pese a que perderá ancho de banda).

    • Las interfaces LAG tienen nombres diversos como Portchannel o Etherchannel.

    • Habitualmente el máximo número de interfaces que pueden meterse en un mismo equipo es de 8.

    • Cada cierto tiempo (1s-30s) se mandan paquetes “Hello” para comprobar que ningún enlace no se ha caído y seguir mandando tráfico por ahí.

    • LACP permite varias formas de balanceo de carga, basándose en el flujo, en el origen o destino de la IP, en la MAC, etc. El número de opciones implementables depende de fabricante.

    • Para que funcione todas las interfaces deben funcionar a la misma velocidad (10M/100M/1G/10G…) y se full dúplex. Si no es así se producirá un aggregation mismatch.

    • Las interfaces que conforman el LAG deben ir conectadas al mismo equipo lógico, es decir, no pueden ir conectados a equipos totalmente independientes física y lógicamente (por ejemplo que no estén en stack).

    • Cada interfaz, en cada extremo puede configurarse de un modo:

      💥 Activo: Envía paquetes LACP para establecer el portchannel.

      💥 Pasivo: Espera la recepción de un paquete LACP para integrar dicha interfaz dentro del portchannel.

    Note

    Al menos debe haber una interfaz entre los dos extremos en modo activo.

    2) Indica la configuración de un Portchannel en Cisco.

    3) Indica la configuración de un Portchannel en Juniper.

    4) Comando en Cisco para determinar una IP destino por cuál de las interfaces de un portchannel viajará.

    5) Comando en Cisco y en Juniper para comprobar el estado de una interfaz portchannel.

    6) Dada una red con un servidor con mucho tráfico, qué sucedería si creamos el portchannel balanceando por MAC destino.

    7) Busca información sobre el protocolo PAgP y MLAG
