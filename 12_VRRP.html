    Alta disponibilidad: HSRP, VRRP y GLBP.¶

    Se trata de una serie de protocolos que tienen como principal finalidad dotar a los equipos de capa 3 (routers o switch multicapa) de una redundancia y permitir una alta disponibilidad. Esto se consigue al comunicar estos equipos de forma que uno haga de maestro/activo y otro/s de respaldo/pasivo.

    Habitualmente para ello se configura a todos los equipos (activo y pasivos) en un mismo grupo (0-255) y además se hará uso de una IP virtual/física, que será la que verán los clientes, y la cual siempre que haya al menos un equipo del grupo activo se encontrará disponible.

    Es importante tener en cuenta, que aunque todos los equipos del grupo tengan la misma IP virtual, a parte tendrán su propia IP física de interfaz.

    HSRP (Hot Standby Router Protocol).¶

    • Es un protocolo propietario de Cisco y es el más antiguo de los 3.

    • Los equipos dentro del grupo HSRP se intercambian paquetes cada cierto tiempo (por defecto 3 segundos) para detectar alguna caída. Estos mensajes se mandan mediante multicast 224.0.0.2 al puerto 1985 UDP. Además se suele especificar un tiempo de espera (holdtime) tras el cual se da por inactivo un router. Este tiempo suele ser por defecto 10 segundos.

    • Para saber qué equipo actúa como maestro se asigna unas prioridades (0-255), siendo 100 el valor por defecto. Aquel equipo con prioridad más alta será el maestro. En caso de tener la misma prioridad, el equipo con IP física de la interfaz más alta será el maestro.

    • Cuando el equipo maestro cae, su prioridad baja a 0 y otro equipo le sustituye. Una vez vuelva el maestro, será de nuevo el router activo.

    • La IP para la gestión del protocolo debe ser una IP virtual. A la IP virtual se le asocia una MAC virtual 0000.5e00.01XX, donde XX es el número de grupo VRRP en hexadecimal.

    VRRP (Virtual Router Redundancy Protocol).¶

    • Es un protocolo prácticamente idéntico a HSRP pero se trata de un protocolo no propietario, por lo que es más habitual su uso.

    • Usa la IP multicast 224.0.0.18 y número de protocolo IP 112.

    • Una de las principales diferencias es que la IP del grupo, puede ser virtual o física, mientras que en HSRP debe ser virtual.

    • La MAC asociada a la IP VRRP es 0000.5e00.01XX y los mensajes “hello” se mandan con un intervalo de 1 segundo por defecto.

    • No soporta balanceo de carga, aunque se podría crear varios grupos y asignar diferentes equipos a diferentes grupos para conseguir un balanceo. No obstante, esto no está recomendado.

    GLBP (Gateway Load Balancing Protocol).¶

    • Protocolo propietario de Cisco que mejora el funcionamiento de HSRP, su principal diferencia frente a HSRP es que puede haber más de un router activos a la vez de tal forma que se pueda realizar un balanceo de carga.

    • Permite configurar pesos para indicar el ancho de banda o tráfico que debe llevar cada interfaz.

    • La prioridad por defecto es 100, igual que en VRRP.

    • Permite hasta 1024 grupos por interfaz y hasta 4 IPs virtuales por grupo.

    • Permite autentificación por contraseña.

    • AVG es el router activo como principal: Active Virtual Gateway.

    • AVF son los routers que conforman el grupo GLBP: Active Virtual Forwarder.

    • Las configuraciones se realizan dentro de la interfaz.

    • El balanceo de cargar funciona de la siguiente forma:

      💥 El AVG reúne todos las MACs de los AVF y cuando un host pregunta por la MAC de su Gateway le da una MAC de un AVF, al siguiente que pregunta le da otra MAC, etc.

    Configuración Ejemplo.¶

    Warning

    Todo se configura dentro de la interfaz donde activaremos el protocolo.

    Configuración ejemplo en ambos routers:

    int Gi0/0
glbp 10 ip 10.0.0.1
glbp 10 authentication text ochoa
glbp 10 priority 200
glbp preempt delay 30
glbp 10 load-balancing host-dependent

    • El 10 de los ejemplos es el identificador del grupo GLBP, es como un nombre.

    • Configurar la IP virtual (Ej: 10.0.0.1). Se usa una IP libre de la misma red física.

    • Establecer una contraseña para unirse al grupo (ochoa).

    • Configurar una prioridad (Ej: 200).

    • Si un router con más prioridad levanta, no tira al otro con menor prioridad que está como maestro hasta que hay una caída. No obstante, podemos forzar que el router con mayor prioridad siempre sea el maestro con la opción de "preempt". Esto tendrá sentido si hay una diferencia de calidad entre el equipamiento que tenemos como maestro/esclavo. Se le puede añadir un temporizador de espera, esto es recomendable por si el router o cable está cayendo y levantando continuamente que no haya un cambio de maestro cada pocos segundos.

    • Crear balanceo de carga. Por defecto es Round Robin.
