    Servidor PPTP.¶

    Vamos a configurar un servicio VPN PPTP en un equipo Mikrotik.

    Crear el pool para clientes VPN.¶

    Primero debemos añadir un POOL, es decir, un rango de direcciones, que se asignará a la gente que se conecte a través de la VPN. Este rango de IPs (pool) debe ser de la misma LAN a la que se va a acceder.

    1. Para configurarlo vamos a “IP” ➡️ “pool”

    2. Limitaremos el pool del DHCP desde la IP .100 a la IP .254. (192.168.88.100-192.168.88.254).

    3. El servicio PPTP tendrá un pool desde la .10 a la .99 (192.168.88.10-192.168.88.99).

    Configurar PPP.¶

    Una vez creado el rango de direcciones que se asignaran a los clientes que se conecten crearemos un perfile PPP. Configuramos PPP porque PPTP usa el protocolo PPP tunelizado. El local address y remote address es que rango de ips se puede asignar al cliente que se conecta, es decir, el pool que hemos creado.

    1. Vamos al menú “PPP” ➡️ “Profiles” ➡️ “Add” (signo del “+”).

    2. Tanto el local address como el remote address son los rangos que se asignaran al cliente que se conecta, por tanto usaremos el pool que creamos en el punto anterior.

    Crear usuarios.¶

    A continuación crearemos un usuario y su contraseña.

    1. “PPP” ➡️ “Secrets” ➡️ “Añadir” (“+”).

    2. En primer lugar vamos a usar una contraseña débil, por ejemplo “salva”.

    3. Importante que el perfil que se le asigne sea el del punto anterior.

    Activar servicio.¶

    Activamos el servidor.

    1. “PPP” ➡️ “Inteface” ➡️ “Pestaña PPTP Server”.

    2. Para activarlo debemos pinchar en “Enabled”.

    Configurar Firewall.¶

    A continuación comprobamos que el firewall permita conexiones PPTP.

    1. Vamos al menú “Firewall” ➡️ “Firewall Rules”

    2. Añadimos en la pestaña “General” que se va a permitir la entrada (input) del protocolo TCP al puerto 1723 (por defecto usado en PPTP). Si hubiéramos cambiado el puerto por defecto de PPTP, obviamente deberíamos modificar aquí también.

    3. Ahora en la pestaña “Action” de esta misma ventana vamos a indicar que cuando detecte este flujo de datos lo va a permitir (accept).

    4. De forma habitual, se debe colocar esta regla siempre por encima (con mayor prioridad), que una regla que deniegue este tráfico.

    Cambiar los puertos.¶

    En caso de querer cambiar los puertos del servicio PPTP, que no lo vamos a hacer en esta práctica:

    1. “Firewall” ➡️ “Service Ports” y doble click sobre el protocolo pptp.

    Problemas de conexión interna.¶

    Una vez conectado, si no puedes hacer ping desde fuera a los equipos de dentro es posible que se deba a que no funciona el protocolo ARP, debemos activar proxy-arp en la interfaz local (LAN).

    1. Para ello vamos a “Interfaces” ➡️ “bridge” y activamos proxy-arp como se muestra en la imagen.

    Comprobación.

    Finalmente podemos a probar a conectarnos desde un cliente, por ejemplo Windows.

    Una vez conectado en el apartado “PPP” ➡️ “Active connections” debería verse los clientes activados y las IPs asignadas.
