    Comandos NAT.¶

    Dado el siguiente esquema vamos a configurar el router para que el PC pueda navegar usando una IP privada "NATEADA".

    Configurar las interfaces.¶

    Levantar las interfaces.

    Por defecto un router viene con las interfaces administrativamente apagadas o tiradas (admin down).

    Por lo que vamos a proceder a levantarlas, tanto la Fa0/0 como la Fa1/0.

    int Fa0/0
no shutdown 

int Fa1/0
no shutdown

    Configurar la IP de las interfaces

    int Fa0/0
description WAN
ip address dhcp

int Fa1/0
description LAN
ip address 10.0.0.1 255.255.255.0

    Configurar NAT.¶

    La idea es que las IPs de nuestra LAN (10.0.0.0/24), salgan por la WAN enmascarada o traducida como una 192.168.1.X/24 (En un caso real sería una red pública).

    Generamos un pool

    En primer lugar vamos a generar un “pool” o un grupo de IPs, en el caso de Cisco lo vamos a hacerlo a través de una lista de acceso.

    Toda IP origen que cumpla esta regla se meterá dentro del NAT para su traducción. En este caso coincide con cualquier IP dentro de la 10.0.0.0/24 que tenemos configurada en nuestra LAN.

    access-list 1 permit 10.0.0.0 0.0.0.255

    WILDCARD

    El 0.0.0.255 se conoce como wildcard y selecciona aquellos bits que coincidan con 0s, es decir, en el ejemplo todo lo que empiece por 10.0.0.X tendrá una coincidencia.

    Configurar Traducción PAT

    Deberemos decirle cuáles serán las IPs que vayan a traducirse (10.0.0.0/24) y por cuales se van a traducir (la configurada en nuestra WAN).

    Las IPs que van a ser traducidas normalmente se conocen como “Inside local” (10.0.0.0/24) y las IPs por las que van a ser traducidas y que van a ser visibles del exterior son “Global local” (WAN).

    Recordemos que “source list 1” hace referencia a la lista de acceso creado en el paso anterior y la interfaz F0/0 es nuestra interfaz WAN.

    ip nat inside source 1 interface Fa0/0 overload

    Configurar NAT dentro de las interfaces

    Ahora vamos a especificar el papel de cada interfaz dentro del NAT.

    Recordamos que inside son las IPs que se van a traducir (habitualmente nuestra LAN) y el nat outside es la IP por la que se traducen (habitualmente nuestra WAN).

    int Fa0/0
ip nat outside

int Fa1/0
ip nat inside

    Comprobaciones.¶

    Lanzar un ping

    Desde el PC probaremos a lanzar un ping, por ejemplo al 8.8.8.8

    Comprobar traducciones

    Desde el router podemos comprobar que se están haciendo traducciones.
