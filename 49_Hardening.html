    Hardening Cisco.¶

    Guía oficial Cisco.

    Configuraciones Recomendadas.¶

    Crear un usuario y contraseña a nivel local con privilegios de nivel 15 (local).

    username admin privilege 15 password password

    Podemos gestionar los usuarios también a través de servidores RADIUS / TACACS+.

    Crear contraseña de “enable”.

    enable secret password

    Cifrar las contraseñas de la configuración.

    service password-encryption

    Se trata de un cifrado débil, no es el mismo que usado para la contraseña de enable.

    Limitar el número de fallo de autentificaciones por usuario. 
    aaa new-model
aaa local authentication attempts max-fail 15
aaa authentication login default local
    Desactivar el password-recovery para evitar que alguien con acceso local al router a través del puerto de consola pueda modificar la contraseña.

    no service password-recovery

    Activar SSH. 
    hostname NOMBRE
ip domain-name Gomariz-Dominio
crypto key generate rsa
line vty 0 15
  transport input ssh
ip ssh time-out 120
ip ssh authentication-retries 3
    Activar SNMP y filtrarlo para que solo conteste a peticiones de la red de mantenimiento. 
    ip access-list standard SNMP-ACL
permit 192.168.123.0 0.0.0.255

#SNMPv2.
snmp-server community Severo8a RO SNMP-ACL

#SNMPv3.
#Severo8a → usuario
  * AuthSevero8a → contraseña de autenticación
  * PrivSevero8a → contraseña de cifrado

snmp-server group SNMPV3-GRP v3 priv access SNMP-ACL
snmp-server user Severo8a SNMPV3-GRP v3 auth sha AuthSevero8a priv aes 128 PrivSevero8a
    Deshabilitar servicios por defecto. 
    #Telnet
line vty 0 15
transport input ssh

#HTTP
no ip http server

#Finger (proporciona información de los usuarios).
no ip finger

#bootp server (Volcado IOS por red).
no ip bootp server

#DNS
no ip domain-lookup

#CDP
no cdp run

#Configuración por red
no service config
    Tirar administrativamente abajo interfaces que no se vayan a utilizar.

    Con el comando shutdown entrando primeramente en la interfaz a suspender.

    Crear un nombre/descripción en aquellas interfaces que se vayan a usar.

    Con el comando description entrando primeramente en la interfaz.

    Activar el registro local (buffered logging). 
    logging buffered informational 
logging buffered 64000
    Configuración de Interfaces. 
    int Gi0/0
#ICMP Redirects: Para no contestar a tráfico que tenga como destino la misma interfaz por la que entra. Es una forma de avisar que existe una ruta más directa.
no ip redirects
#ICMP unreachables: Para que cuando se corte algún tráfico no esté contestando con ICMP Unreachable.
no ip unreachables
#Proxy ARP: Permite mandar paquetes ARP más allá de tu red, redirigidos por el router.
ip arp proxy disable
    Activa DHCP snooping y DAI (Dynamic ARP Inspection) para evitar ARP spoofing. 
    #Se activa a nivel general.
ip dhcp snooping

int Gi0/X
# SOLO SE APLICA EN INTERFACES CON UN SERVIDOR O EN UNA TRONCAL
ip dhcp snooping trust
ip arp inspection vlan <vlan-range>
ip arp inspection limit rate <rate_value> burst interval <interval_value>
    Configuración Unicast RPF (IP Spoofing). 
    int Gi0/0
#La ruta de retorno hacia la IP origen existe en la tabla de routing y es coherente con la interfaz por la que llegó el paquete.
ip verify unicast source reachable-via rx

    Listas de Accesos.¶

    En cisco hay dos tipos de ACL (Estándar y Extendidas).

    • ACL Estándar:

      🔥 Tienen el ID entre 0-99 o 1300 - 1999.

      🔥 Solo filtran por IP origen.

      🔥 Se pueden añadir nuevas reglas al final, pero no en medio.

      🔥 Tienen un deniega todo implícito.

      🔥 Posteriormente hay que aplicarla en la interfaz.

    Ejemplo ACL estándar. 
    access-list 10 permit 192.168.1.0 0.0.0.255
#No haría falta realmente porque está implícito.
access-list 10 deny any

interface GigabitEthernet0/0
ip access-group 10 in

    • ACL Extendida:

      🔥 Pueden filtrar por host/red origen/destino, puertos, protocolos (tcp/udp/ip).

      🔥 Se pueden añadir/eliminar reglas en medio de la ACL.

      🔥 Tienen un deniega todo implícito.

      🔥 Posteriormente hay que aplicarla en la interfaz.

    Ejemplo ACL Extendida. 
    ip access-list extended FIREWALL-LAN

remark 1. Permitir DNS hacia servidor interno
permit udp 192.168.24.0 0.0.0.255 host 10.10.10.20 eq 53

remark 2. Permitir HTTP hacia servidor web interno
permit tcp 192.168.24.0 0.0.0.255 host 10.10.10.10 eq 80

remark 3. Permitir navegación web hacia Internet
permit tcp 192.168.24.0 0.0.0.255 any eq 80
permit tcp 192.168.24.0 0.0.0.255 any eq 443

remark 4. Permitir SSH solo desde host administrador
permit tcp host 192.168.24.50 host 10.10.10.10 eq 22

remark 5. Permitir tráfico TCP de retorno (Internet → LAN)
permit tcp any 192.168.24.0 0.0.0.255 established

remark 6. Denegar todo lo demás y registrar
deny ip any any log

#Aplicar el filtro.
interface GigabitEthernet0/0
  ip access-group FIREWALL-LAN in

    Deniega tráfico ICMP excepto desde la red de mantenimiento.¶

    access-list 101 permit icmp 192.168.123.0 0.0.0.255 any
access-list 101 deny icmp any any
access-list 101 permit ip any any

interface Gi0/0
ip access-group 101 in

    Evitar el Spoofing con una ACL que permita el tráfico a tu interfaz únicamente de las redes que tienes configuradas en la LAN.¶

    access-list 103 permit ip 192.168.123.0 0.0.0.255 any
access-list 103 permit ip 10.0.0.0 0.0.0.255 any
deny ip any any log
